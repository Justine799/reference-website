Relative URLs and paths

Paths: Syntax we write to link and find files and folders on our computer.

URL: The address of a web page; the URL makes it possible for a computer to locate and open a web page on a different computer on the Internet. When it comes to paths for a website they also turn into URLs (paths = URLs when making websites).

Folders

The root folder:

The primary folder for the website (our GitHub repository). It's where our homepage resides, our index.html, filed DIRECTLY inside our root folder.

Paths in the same folder:

When creating a path within the same folder, we only need to specify the filename of that folder.

  • Ex: From index.html to plesiosaurs.html within same folder: path = plesiosaurs.html
  • Even if we’re inside a folder and need to get to another file in that same folder, we just write the name of it (write the name of the destination file). We can write "./" and get the same effect. The "./" is implicit, meaning it will be added if we don’t add it ourselves.

Moving into folders:

To write a path that moves into a folder, we specify the folder name, followed by a forward slash, then the file name.

  • Ex: From index.html to tyrannosaurus.html (in dinosaurs folder):path = dinosaurs/tyrannosaurus.html
  • Sub-folder index: If a folder has its own index.html file, we don’t need to specify that in the path

Moving out of folders:

When moving out of folders, we use ". . /", then follow all the rules above.

  • Ex: From apatosaurus.html to pterosaurs.html: path = . . / pterosaurs.html
  • Ex #2: From apatosaurus.html to logo.svg: path = . . / images/logo.svg
  • If you need to get out of multiple folders, just add more:
    • Out of 2 folders = . . / . . /
    • Out of 3 folders = . . / . . / . . /

Relative vs. absolute paths:

Relative paths:

  • All of the paths we learned above are all relative to the location you’re currently in - called document relative.
  • If the path starts with nothing, it means “start where I am and go from here”.
  • If the path starts with ". . /", it means “go up one folder from where I am and go from here”.

Absolute paths:

  • Absolute paths are in relation to the top-level internet and URLs.
  • If we start a path with a protocol (http(s)), it means “start at the top level of the Internet, replacing everything, and work from there”.
  • If our website was hosted at the domain “prehistoric-animals.com”, we’d get the following paths:
    • https://prehistoric-animals.com/
    • https:// prehistoric-animals.com/plesiosaurs.html
    • Etc.

Protocol relative:

  • If we don’t want to write the protocol (https) we can just leave it off and the browser will automatically add it for us.
  • If we start a path with a / / , it means “start immediately after the protocol, replacing all domains, and work from there”.

Root relative:

  • If we only want to write paths within our domain but have more absolute-like paths, we can use root-relative paths.
  • If we start a path with /, it means “start at the root domain and work from there”

When do we use this stuff?:

  1. In HTML links (href)
  2. In image tags (src)
  3. When linking CSS (href)
  4. When using background-image in CSS
