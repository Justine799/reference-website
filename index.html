Naming conventions

There are three rules to follow when naming files on the web:

  1. All lowercase
  2. No spaces
  3. Only letters, numbers and dashes

Components of creating a website

  • HTML (content, structure, hierarchy of information)
  • CSS (Cascading Style Sheets - appearance, layout)
  • JavaScript (behavioral elements that appear when used by people such as buttons, filtering, search, etc)

What does HTML mean?

Also called Hypertext Markup Language, HTML is a coding language used to describe the content of websites and display content

How to create repositories on Github

A repository is a place that contains all project files and each files's revision history.

Steps to follow to create a repository:

  1. Go on the GitHub website and sign-in to your account
  2. Create a new repository and include a README.file (title with hyphens, make it public, set i to main branch)
  3. Clone down repository to your local environment (destop or computer device) using Github Desktop
  4. Go on text editor (Visual Studio Code) and open corresponding folder
  5. Create an index.html file
  6. Commit work for a first time using an acceptable commit message. Make sure to use an imperative word when creating commit messages.
  7. Push work up to Github (push to origin)
  8. Update index.html file to include html boilerplate
  9. Update index.html file with other information and commit each time to push work up and save progress

Tag syntax

Headers

H1: Usually used for the title

  • (<)h1(>) Here goes the body copy (<)/h1(>)
  • Main header
  • Can only be entered once

H2 to H6

  • (<)h2(>) (<)h3(>) (<)h4(>) (<)h5(>) (<)h6(>)
  • H2 is the subheader of h1, h3 is the subheader of h2, so on and so forth.

Paragraphs

  • (<)p(>) Here goes the paragraph text(<)/p(>)

Lists

Unordered lists

  • (<)ul(>)
  • The order of the items doesn't matter
  • (<)li(>) is nested inside of unordered list
  • You can have as many bullets as you want

Ordered lists

  • (<)ol(>)
  • The order of the items matter
  • (<)li(>) is nested inside of ordered list
  • You can have as many numbers as you want

Description lists

  • (<)dl(>)
  • Items that are categorized and used to describe a specific term
  • (<)dt(>) is nested inside, which is the name or description of the term
  • (<)dd(>), which is the description of term, is nested inside of (<)dt(>)

Quotes and citations

  • (<)q(>): Used to mark up quotes embedded in other elements, like a paragraph
  • (<)blockquote(>): Larger, more decorative way of displaying a quote
  • (<)cite(>): Marking the source of the quote

Phrasing elements

  • (<)em(>): Used to display emphasis (italicized)
  • (<)strong(>): Used to dipsplay lots of emphasis, strong importance or urgency (bold)
  • (<)i(>): Used to represent a word in another language, a technical term, title (ex; chinese character, italian word, etc)
  • (<)b(>): Keyword (bold)

Links

How to create a hyperlink:

  • (<)a href=“”(>) Link text (<)/a(>)
  • The anchor tag : The opening (<)a(>) tells us where the link should start and the closing (<)/a(>) indicates where the link ends.
    • To create a hyperlink, you must put the URL (link) between the quotation marks, and enter the clickable text before the closing anchor tag (where "Link text" is situated).

How to add an image:

  • (<)img src“” alt=“” /(>)
  • Images in websites are not embedded in the website, but linked by the HTML document.
  • Images need two attributes to specify extra details about the image: the location of the image (image source) and the alternative content (description of content - content that would appear in case the image doesn't appear).
  • You must add the URL (link) in the first quotation marks, and the description of the content in the second quotation marks
  • Self-closing tag

Other

  • (<)small(>): Decrease text size

How to structure content on the page

Generic elements

  • Meaningless content
  • (<)div(>) = Block elements used to group elements together (adjust layout, CSS or behaviour JavaScript)
  • (<)span(>) = An inline container used to mark up a part of a text, or part of a document (wrap around a sentence, a few words, bold it, change font, make it abig bigger)

Attributes

  • HTML attributes provide additional information about HTML elements (extra info to make something work or give more info to later use elsewhere)
  • Allows us to target any HTML element within CSS or JavaScript
  • (<)a href=""(>) Link text (<)/a(>)
  • (<)p class=""(>) You can use a class multiple times throughout your HTML and attach it to different type of elements
  • (<)p id=""(>) Only allowed to use one unique identifier once in each HTML page

Content structure

Organizing our HTML elements content within typical regions of a web page.

Headers

Introductory elements of the header:

  • Company logo
  • Title
  • Site name
  • Main naviguation
  • Also used within sections and articles to introduce the content for headings, subheadings, author, etc
  • (<)header(>)(<)/header(>)

    • Main

    • Content unique to each page, which means it's only used once on a page
    • Tells the browser that this is where the unique content is
    • (<)main(>)(<)/main(>)

    Footer

    Elements of the footer:

    • Copyright information
    • Site info
    • Company address
    • Company second website
    • Terms of services
    • Columns of links
    • Extra info about the company
    • (<)footer(>) (<)/footer(>)

    Used to wrap around and group related pieces of content. Usually, each section has its own heading.

    • (<)section(>) (<)/section(>)

    Article

    Represents a self-contained composition in a web page, which is intended to be independently distributable or reusable.

    • Blog post
    • Product card
    • Interactive widget
    • (<)article(>)(<)/article(>)

    Represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

    • (<)nav(>)(<)/nav(>)

    Aside

    Represents a portion of a document whose content is only indirectly related to the document's main content (this content could be represented as a sidebar or in a section underneath the website - it is not in the Main tags)

    • (<)aside(>)(<)/aside(>)

    Accessibility

    Web accessibility means that everyone, including users with visual, auditive, cognitive or physical disabilities will have an ideal experience and can access the information equally.

    WCAG principles

    All websites must comply with the Web Content Accessibility Guidelines 2.0. There are four principles that they have to follow:

    1. Perceivable (must be able to discern content regardless of senses)
    2. Operable (must be able to find, navigate and interact with the web content in multiple ways)
    3. Understandable (the content must be understandable to its users)
    4. Robust (must be developed using well-adapted web standards accessible across all browsers and assistive technologies)

    Ways of making a website more accessible

    Visual impairments:

    1. Allow text to be resized
    2. Have good contrast in colours
    3. Website must work well with screen readers
    4. Use proper alt attributes

    Physical impairments:

    1. Make website keyboard accessible - ensure users have an alternative way to access information
    2. Increase hit areas of links and buttons

    Auditory impairments:

    1. Provide text captions and subtitles during audios or videos on the website
    2. Users may need closed caption while in a noisy room or when the language spoken is not their first language

    Cognitive impairments:

    1. Make it clear where in the website the user is
    2. Organize your content correctly
    3. Clearly mark links
    4. Use plain, simple language

    Landmark roles

    Landmarks help assistive technology users orient themselves within a page and helps with the navigation of the website (can jump to various sections of a page).

    Headers:

    • (<)header role= “banner”(>)
    • Is used only once at the top of the page, and added to the primary header.

    Navigation:

    • (<)nav role=”navigation”(>)
    • Is added to the primary navigation (<)nav(>)

    Main:

    • (<)main role=”main”(>)
    • Defining an element as being the main content of the page
    • Added to the (<)main(>)

    Aside:

    • (<)aside role=”complimentary”(>)
    • Defining an element as content related to the primary content.

    Footer:

    • (<)footer role=”contentinfo”(>)
    • Defining an element as information about the content copyright, terms, privacy policy.

    Labeling links

    Adding aria-label attribute provides the user with a non-visual label that will be announced by screen readers.

    • (<)a href=”#” aria-label=”Description of what the user will click through to would go here”>Read more(<)/a(>)

    Image descriptions

    Alt attribute has two purposes:

    1. To describe the picture if it doesn't download
    2. To describe a picture to someone with visual disabilities, and who might use assistive technology

    Skip links

    Skip links are placed at the very top of your page and offer users a way to jump to specific content. It also gives people who use keyboards access to jump to different specific areas on a page.

    • Example : (<)li(>)(<)a href="#nav"(>)Name of specific area on the page(<)/a)>)(<)/li(>)
    • Afterwards, insert role in appropriate section : (<)nav role="navigation" id="nav"(>)(<)/nav(>)
